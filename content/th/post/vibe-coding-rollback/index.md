วิศวกร A เปิด Cursor และบอก AI ให้แก้ bug AI ทำเสร็จ code รันไม่ได้ เขาบอก AI ให้แก้อีกครั้ง AI ไปแตะไฟล์ที่สาม ยังพังอยู่ มันแก้ไฟล์ที่ห้า ตอนนี้วิศวกร A ไม่แน่ใจแล้วว่า AI เปลี่ยนไฟล์อะไรบ้าง

ตอนนี้คุณคงคิดว่า: หยุด กลับไปสู่สถานะที่อย่างน้อยรันได้เมื่อสักครู่ที่ผ่านมา

ปัญหาคือ: **คุณจะรู้ได้ยังไงว่าเวอร์ชันไหนคืออันที่รันได้?**

## ช่วงเวลาที่ AI ทำเกินหน้าตาเป็นยังไง? {#ai-overshoot}

คุณกำลัง vibe coding คุณส่งเป้าหมายให้ AI AI เขียนเป็นชุด

รัน OK

รอบถัดไป คุณบอก "เพิ่มอีกฟีเจอร์" AI แตะ 3 ไฟล์ รัน — error

คุณบอก "แก้ error นั้น" AI แตะ 5 ไฟล์ แก้ config เพิ่ม helper function ที่คุณไม่เคยขอ รัน — error เพิ่มขึ้น

![หน้าต่างแชทของ AI agent กับจำนวนไฟล์ที่เปลี่ยนจริงบนคอมพิวเตอร์ของคุณ](image-1.svg)

AI ยังคงแก้สิ่งต่างๆ ด้วยความมั่นใจ **มันจะไม่ยอมบอกเองว่า "ฉันอาจจะทำพังไปแล้ว"**

ความจำของมันคือแค่ context window ปัจจุบัน **มันไม่รู้ว่า 5 prompt ที่แล้ว code ของคุณยังโอเค** แต่ไฟล์บนคอมพิวเตอร์ของคุณรู้ ตราบใดที่มีใครบางคนจำได้

## หนึ่งการกระทำ: เปิด Timeline คลิกจุดที่ทำงานได้ล่าสุด {#one-action}

### ขั้น 1: เปิด Keeply Timeline

แท็บแรกใน sidebar ทางซ้าย คุณจะเห็นทุกการเปลี่ยนแปลงวันนี้ เรียงตามเวลา

### ขั้น 2: หาจุดสุดท้ายที่ code "ยังรันได้"

แต่ละรายการบน Timeline เป็นจุดบันทึกอัตโนมัติของ Keeply (save point) หรือช่วงเวลาที่คุณ mark ด้วยมือ เปิดแต่ละจุดดูการเปลี่ยนแปลงข้างใน หาเวอร์ชันที่คุณจำได้ว่า "ทดสอบโอเคตอนนั้น"

ปกติ 30-60 นาทีที่แล้ว การทดสอบครั้งสุดท้ายก่อน AI เริ่มเฉไฉ

![Keeply Timeline ขยาย: file note แต่ละอันแสดง timestamp + บรรทัดที่เปลี่ยน + บันทึกการทดสอบของคุณก่อนหน้า](image-2.svg)

### ขั้น 3: คลิกขวารายการนั้น เลือก Restore (กู้คืน)

ทั้งโฟลเดอร์กลับไปสู่จุดเวลานั้นใน 30 วินาที **ทุกไฟล์ ทั้ง directory tree ทุก config — กลับไปด้วยกันหมด** ไม่ใช่แค่ไฟล์เดียว

นั่นรวม helper function ที่ AI แอบใส่เข้ามา config ที่มันแก้ .env ที่มันไม่ควรแตะ **ทั้งหมดกู้คืนกลับไป**

แล้วคุณรัน มันใช้ได้

![ก่อนกับหลังการกู้คืน: file tree + ไฟเขียวจากการรัน test](image-3.svg)

ทั้งกระบวนการใช้เวลาน้อยกว่าหนึ่งนาที **คุณไม่ต้องจำว่า AI แตะไฟล์อะไรบ้าง Keeply จำให้หมดแล้ว**

## ทำไม AI ถึงไม่กู้คืนตัวเอง {#ai-doesnt-rollback}

AI agent ถูกออกแบบมาเพื่อ **เดินหน้า** มันรับ prompt ผลิตการแก้ไข มันจะไม่หยุดมองย้อนกลับและถาม "รอบที่แล้วทำให้โปรเจกต์แย่ลงหรือเปล่า"

ความรับผิดชอบนั้นไม่ได้อยู่กับ AI มันคือข้อจำกัดเชิงสถาปัตยกรรม

ความรับผิดชอบอยู่กับคุณ: **คุณต้องมีตาข่ายนิรภัยรันอยู่เบื้องหลัง** ปล่อยให้ AI วิ่งไปไกลแค่ไหนก็ได้ เพราะคุณดึงกลับมาได้

Keeply ไม่ได้มาแทนส่วนที่คุณเขียน code มันมาเพื่อให้ตอนคุณ vibe coding คุณไม่ต้องพึ่งความจำเพื่อย้อนกลับ ความจำแพ้ความเร็วที่ AI แก้ไฟล์
