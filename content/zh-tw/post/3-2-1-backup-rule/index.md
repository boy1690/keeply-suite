# 資料備份的 3-2-1 原則：20 年了，2026 還夠用嗎？

> 3-2-1 規則 20 年沒變，但你今天怕的事跟 2005 年不一樣了。

2005 年，攝影師 **Peter Krogh** 為自己訂下備份規則：3 份檔案、2 種媒介、1 份異地。他防的是磁帶腐壞、硬碟摔到地上、機房失火。

20 年後的你，怕的是 **⌘+S 多按了一次**。

3-2-1 規則沒變過——但你的痛點換了。

## 重點

**3-2-1 備份原則** 是必要的：三份檔案、兩種媒介、一份異地，防的是硬碟損毀、機房失火這類災難。但它從設計就沒處理 **operator-error**。你自己覆蓋、同事改錯、雲端同步把錯的版本傳到三個位置，3-2-1 都救不了。本文拆解 3-2-1 防什麼、不防什麼，以及那個沒被處理的層需要什麼。

## 3-2-1 規則到底在說什麼？

3-2-1 是 Peter Krogh 在 2005 年[《The DAM Book》](https://www.oreilly.com/library/view/the-dam-book/9780596008550/)（O'Reilly Media）訂下的備份規則：

- **3 份**檔案：原檔加 2 份備份
- **2 種**儲存媒介：例如本機硬碟加雲端，或 NAS 加外接 SSD
- **1 份**存放異地（其中一份放在物理上不同地點）

當時主流媒介是磁帶、CD/DVD、機械硬碟，物理損毀率高、媒介老化快。3-2-1 的設計目的很清楚：讓任何單一硬體故障、媒介老化、機房災難都救不死你的檔案。

## 3-2-1 防什麼，不防什麼？

要看 3-2-1 守不守得住，先看「會丟檔」的劇本長什麼樣。

| 場景 | 3-2-1 救得了嗎？ | 為什麼 |
| --- | :---: | --- |
| 硬碟摔壞 | ✅ | 3 份備份在不同媒介 |
| 機房失火 | ✅ | 1 份在異地 |
| 勒索軟體加密 | ✅（異地份未中招） | 異地隔離 |
| **你自己覆蓋掉版本** | ❌ | 3 份都會被同步成新版 |
| **同事改錯共用資料夾** | ❌ | 同上 |
| **找 3 個月前的舊版** | ❌ | 3-2-1 不是版本歷史 |

對啊，這就是讓人煩的地方。3-2-1 防的是「檔案不見了」。它沒處理「檔案還在但變錯了」。

## 為什麼你做了 3-2-1 還是丟檔？

這裡要拆一個 20 年沒人明講的盲點：**「3 份備份」的「3」是空間冗餘，不是時間冗餘。**

2005 年硬碟壽命短、媒介易壞，多放幾份是為了對抗物理損毀。這時候「3」是合理解。

2026 年硬碟可靠、雲端即時同步，「3」變成了什麼？變成同一個錯誤被即時複製到 3 個位置。

A 先生是設計師。週一早上 10:32，客戶打電話來要 3 個月前簽好的提案版本。他打開 NAS，看見 12 個版本，3 份雲端備份每份都是現在最新的。

但他不要最新。他要 3 個月前那一版。

等等，這還不是最糟的。他在備份做完之後，才發現「最新」不是他要的。也就是說 3-2-1 把錯的版本盡責地保護了 3 份。

## 3-2-1 + 版本歷史，能不能一套搞定？

可以。[Keeply](https://keeply.work) 把 3-2-1 內建為位置層：

- **本機工作副本**：你電腦上的工作版本（對應 3-2-1 的「1 份」）
- **正本（專案位置）**：NAS 或雲端的 canonical 存放處（對應「2 種媒介」）
- **備援位置**：整個專案同步到另一個位置（對應「1 份異地」）

加上 git 級別的版本歷史，以及 Release 凍結機制。一套工具，三層保護。

Keeply 不替你決定備援位置該放哪。如果你把本機和備援都放在同一個辦公室，火災一起燒。任何工具都救不了這種事。「異地」原則仍然要你自己決定。

但你不需要兩個工具：一個管空間冗餘，一個管時間冗餘。一套 Keeply，從本機到備援、從這秒到上週，都看得見、找得回。

## 常見問題

**Q1: 3-2-1 規則和 4-2-1-1-0 規則的差別？**

4-2-1-1-0 是 3-2-1 的延伸：多 1 份不可變備份，加上 0 個錯誤驗證。本質仍是空間冗餘，**不解版本歷史問題**。

**Q2: 雲端備份算 3-2-1 的「異地」嗎？**

算。但 iCloud、OneDrive、Google Drive 是同步（sync）不是備份（backup）。你刪除或覆蓋會即時同步到雲端，**不防 operator-error**。

**Q3: NAS 算 2 種媒介嗎？**

NAS 加本機硬碟可算 2 種媒介。但 RAID 不算備份。RAID 防的是硬碟壞掉，不防你自己刪錯。

**Q4: Keeply 已經是 3-2-1 嗎？**

是。Keeply 把 3-2-1 內建為位置層（本機工作副本 + 正本 + 備援位置），加上版本歷史和 Release 凍結。一套工具同時做三層。

**Q5: 個人工作者也需要 3-2-1 嗎？**

看你的檔案重要性。丟了會痛就需要。判斷標準是「丟了會不會痛」，跟個人或企業無關。
