## Почему появляется синий экран (это не проблема Keeply) {#why-smartscreen}

Этот экран называется [SmartScreen](https://learn.microsoft.com/en-us/windows/security/operating-system-security/virus-and-threat-protection/microsoft-defender-smartscreen/). Он не решает «вредоносная ли это программа» — он решает «достаточно ли людей уже её использовали».

Думай так: новый ресторан без отзывов в Google — это не плохая еда. Это просто еда, которую никто пока не оценил.

SmartScreen относится к новому ПО так же. Он строит доверие через **объём загрузок + время**, и каждый новый релиз снова проходит этот период наблюдения. Keeply сталкивается с этим при каждом обновлении. Никакого отношения к безопасности самой программы это не имеет.

Так почему же это пугает людей? Потому что экран даёт только огромную кнопку «Не запускать». Чтобы всё-таки запустить, нужно щёлкнуть крошечную ссылку **Подробнее** сбоку. Визуально это читается не как уведомление — это читается как стена.

Но возиться с этим не нужно. **Keeply опубликован в [официальном репозитории пакетов winget от Microsoft](https://github.com/microsoft/winget-pkgs)**, и этот путь предупреждение не вызывает вообще.

Так что суть не в том, как обойти предупреждение. А в том, как пойти путём, где предупреждение не появляется.

![Предупреждение Windows SmartScreen, маленькая ссылка «Подробнее» обведена](fig-smartscreen-warning.svg)

## Три пути — выбери, что подходит тебе {#three-paths}

| Путь | Подходит, если ты | Время | Синий экран? |
| --- | --- | --- | --- |
| **A. Команда winget** (Windows) | не против вставить одну строку в PowerShell | 2 мин | Нет |
| **B. Официальная загрузка .exe** (Windows) | не хочешь открывать чёрный терминал | 5 мин | Да — мы проведём |
| **C. Официальная загрузка .dmg** (macOS) | работаешь на Mac | 3 мин | Нет, но нужен правый клик |

Выбрал? Перейди в нужный раздел. Остальные пропусти.

## Windows путь 1 — одна команда winget (рекомендуется) {#path-winget}

**winget** — встроенный в Windows «менеджер пакетов», по сути Microsoft Store, но из командной строки. Он встроен в Windows начиная с версии 10 1809. Ничего дополнительно ставить не нужно.

Открой PowerShell (поиск «PowerShell» в меню Пуск), вставь эту строку, нажми Enter:

```powershell
winget install Boy1690.Keeply
```

![PowerShell выполняет winget — загрузка и установка завершаются примерно за 30 секунд](fig-powershell-winget.svg)

Около 30 секунд — и готово. Никакого синего экрана. Никакого мелкого шрифта «Подробнее».

Почему этот путь такой чистый? Потому что чтобы вообще попасть в winget, Keeply должен пройти [официальную проверку Microsoft на GitHub](https://github.com/microsoft/winget-pkgs): они проверяют источник установщика, подписи файлов и поведение установки. Релиз выходит, только когда всё проходит.

Иначе говоря: когда ты запускаешь эту команду, Microsoft уже сделал за тебя один раунд проверки. Проверка SmartScreen на этом пути избыточна, поэтому она просто не появляется.

Короткий путь и доверенный путь — в одной строке.

## Windows путь 2 — скачать .exe {#path-exe}

Не хочешь трогать PowerShell? Хорошо. Зайди на keeply.work, нажми скачать, возьми `.exe`, дважды кликни.

Появится синий экран SmartScreen. **Это нормально** ([почему — см. выше](#why-smartscreen)). Чтобы продолжить:

1. Кликни **Подробнее** (маленький подчёркнутый текст на предупреждении)
2. Появится кнопка **Запустить всё равно**
3. Кликни. Дальше всё делает установщик.

![После клика на «Подробнее» рядом с «Не запускать» появляется кнопка «Запустить всё равно»](fig-smartscreen-run-anyway.svg)

Весь обходной путь добавляет максимум 3 минуты — большая часть из которых психологическая, не реальные клики. Дальше этот путь и путь 1 сходятся.

## Установка на macOS — шаг с правым кликом, который нельзя пропустить {#path-macos}

На Mac синего экрана нет. Но дважды кликнуть при первом запуске не получится — [macOS Gatekeeper](https://support.apple.com/en-us/102445) заблокирует.

Правильный порядок:

1. Скачай `.dmg`, перетащи Keeply в папку «Программы»
2. Открой «Программы», найди Keeply
3. **Правый клик → Открыть** (не двойной клик)

   ![Контекстное меню Finder в macOS с подсвеченным «Открыть» сверху](fig-macos-rightclick.svg)

4. Появится диалог — нажми «Открыть»

   ![Диалог подтверждения macOS с подсвеченной кнопкой «Открыть»](fig-gatekeeper-dialog.svg)

Всё. **Только первый запуск требует этого** — потом двойной клик работает как обычно.

Почему этот обход в первый раз? Gatekeeper блокирует двойной клик для любого приложения, нотаризации которого он ещё не видел. Правый клик → Открыть — способ Apple сказать «я знаю, что устанавливаю, пропусти меня».

Это не особенность Keeply. Любое новое приложение Mac, которого раньше не было на твоей машине, ведёт себя так же при первом запуске.

## После установки — закинь свой первый проект {#first-project}

Установлено — это ещё не сделано. Твой первый проект, защищённый в тот же день, — вот это сделано.

Открой Keeply, нажми **Новый проект**, выбери папку, в которой ты сейчас активно работаешь.

**Что закинуть первым**: то, что ты сейчас держишь в руках, что тебе нельзя потерять и что ты постоянно правишь. Презентация, договор, дизайн-файл, питч — что угодно подойдёт. Не выбирай папку, к которой ты не прикасался полгода. Ценность той папки в архивации, не в защите. Это другая история.

Первый скан занимает 1–2 минуты. После этого Keeply следит за папкой в фоне и **записывает версии автоматически по мере того, как ты сохраняешь**. Никакой кнопки «контрольная точка» жать не нужно.

Гипотетический, но типичный пример: дизайнер закидывает свою папку с питчем за второй квартал сразу после установки. Первый скан занимает 2 минуты. Через три дня он понимает, что в субботу неправильно поменял цвет логотипа — поднять предыдущую версию из истории занимает 20 секунд.

Те, кто использует первый проект в день установки, остаются гораздо дольше, чем те, кто откладывает на неделю.

## Застрял? 5 типичных ошибок {#troubleshoot}

| Симптом | Решение |
| --- | --- |
| Команда `winget` не найдена | Значит, в твоей Windows ещё нет App Installer. Используй путь 2 (скачать .exe) — не борись |
| Win 11 говорит «нужны права администратора» | Открой PowerShell заново через **Запуск от имени администратора** |
| Mac пишет «не может быть открыто, так как от неустановленного разработчика» | Правый клик → Открыть (не двойной клик). См. раздел про macOS выше |
| Корпоративная сеть блокирует загрузку | Используй команду winget — она идёт через CDN Microsoft и обычно проходит |
| Установлено, но не открывается | Перезагрузи. Не помогло? Напиши на [support@keeply.work](mailto:support@keeply.work) |

## Что стоит запомнить

Одно:

Тебе не нужно обходить предупреждение. Тебе просто нужно пойти путём winget, где предупреждение не появляется.
