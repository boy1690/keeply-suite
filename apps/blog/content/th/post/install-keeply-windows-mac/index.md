## ทำไมหน้าจอน้ำเงินถึงโผล่ (ไม่ใช่ปัญหาของ Keeply) {#why-smartscreen}

หน้าจอนั้นเรียกว่า [SmartScreen](https://learn.microsoft.com/en-us/windows/security/operating-system-security/virus-and-threat-protection/microsoft-defender-smartscreen/) มันไม่ได้ตัดสินว่า "ซอฟต์แวร์นี้อันตรายหรือเปล่า" — มันตัดสินว่า "มีคนใช้มากพอแล้วหรือยัง"

ลองคิดแบบนี้: ร้านอาหารใหม่ที่ยังไม่มีรีวิว Google ไม่ได้แปลว่าอาหารแย่ แค่ยังไม่มีใครให้คะแนน

SmartScreen ปฏิบัติกับซอฟต์แวร์ใหม่แบบเดียวกัน มันสร้างความน่าเชื่อถือด้วย **ปริมาณการดาวน์โหลด + เวลา** และทุก release ใหม่ก็ผ่านช่วงสังเกตการณ์นี้อีกครั้ง Keeply เจอแบบนี้ทุกครั้งที่ปล่อย update ไม่เกี่ยวกับว่าตัวซอฟต์แวร์ปลอดภัยหรือไม่เลย

แล้วทำไมมันทำให้คนกลัว? เพราะหน้าจอให้แค่ปุ่ม "Don't run" ขนาดยักษ์ ถ้าจะรันต่อ ต้องคลิกลิงก์เล็กๆ ชื่อ **More info** ที่อยู่ด้านข้าง ทางสายตามันไม่ได้อ่านว่าเป็นการแจ้งเตือน — มันอ่านเหมือนกำแพง

แต่คุณไม่ต้องไปจัดการกับมัน **Keeply ถูกเผยแพร่ใน [Microsoft's winget package repo](https://github.com/microsoft/winget-pkgs)** และเส้นทางนั้นไม่ trigger คำเตือนเลย

ดังนั้นประเด็นไม่ใช่วิธีข้ามคำเตือน แต่คือการเลือกเส้นทางที่คำเตือนไม่เคยปรากฏ

![หน้าจอ Windows SmartScreen warning โดยมีลิงก์เล็กๆ "More info" ถูกวงไว้](fig-smartscreen-warning.svg)

## สามเส้นทาง — เลือกอันที่เข้ากับคุณ {#three-paths}

| เส้นทาง | เหมาะถ้าคุณ | เวลา | หน้าจอน้ำเงิน? |
| --- | --- | --- | --- |
| **A. คำสั่ง winget** (Windows) | ไม่รังเกียจที่จะแปะหนึ่งบรรทัดลงใน PowerShell | 2 นาที | ไม่มี |
| **B. ดาวน์โหลด .exe ทางการ** (Windows) | ไม่อยากเปิด terminal ดำๆ | 5 นาที | มี — เราจะพาคุณผ่านมัน |
| **C. ดาวน์โหลด .dmg ทางการ** (macOS) | ใช้ Mac | 3 นาที | ไม่มี แต่ต้องคลิกขวา |

เลือกแล้ว? กระโดดไปที่หัวข้อที่ตรงกัน ข้ามอันอื่น

## Windows เส้นทาง 1 — คำสั่ง winget เดียว (แนะนำ) {#path-winget}

**winget** คือ "package manager" ที่ Windows มีมาให้ — โดยพื้นฐานเหมือน Microsoft Store แต่เป็น command line มันถูกฝังอยู่ใน Windows ตั้งแต่เวอร์ชัน 10 1809 คุณไม่ต้องติดตั้งอะไรเพิ่ม

เปิด PowerShell (ค้น "PowerShell" ในเมนู Start) แปะบรรทัดนี้ กด Enter:

```powershell
winget install Boy1690.Keeply
```

![PowerShell รัน winget — ดาวน์โหลดและติดตั้งเสร็จในประมาณ 30 วินาที](fig-powershell-winget.svg)

ประมาณ 30 วินาทีก็เสร็จ ไม่มีหน้าจอน้ำเงิน ไม่มีตัวเล็กๆ ของ "More info"

ทำไมเส้นทางนี้สะอาดขนาดนี้? เพราะการจะอยู่ใน winget ได้เลย Keeply ต้องผ่าน [การรีวิวอย่างเป็นทางการของ Microsoft บน GitHub](https://github.com/microsoft/winget-pkgs): พวกเขาตรวจ source ของ installer, ลายเซ็นไฟล์, และพฤติกรรมการติดตั้ง มันออกได้ก็ต่อเมื่อทุกอย่างผ่าน

พูดอีกอย่าง: ตอนที่คุณรันคำสั่งนั้น Microsoft ได้ทำการตรวจสอบรอบหนึ่งให้คุณแล้ว การตรวจของ SmartScreen ในเส้นทางนี้ซ้ำซ้อน เลยไม่ปรากฏ

เส้นทางสั้นและเส้นทางความน่าเชื่อถือ ในบรรทัดเดียว

## Windows เส้นทาง 2 — ดาวน์โหลด .exe {#path-exe}

ไม่อยากแตะ PowerShell? ก็ได้ ไปที่ keeply.work คลิกดาวน์โหลด หยิบ `.exe` ดับเบิลคลิก

หน้าจอน้ำเงินของ SmartScreen จะเด้งขึ้น **นั่นเป็นเรื่องปกติ** ([ทำไม ดูข้างบน](#why-smartscreen)) เพื่อไปต่อ:

1. คลิก **More info** (ตัวอักษรขีดเส้นใต้เล็กๆ บนคำเตือน)
2. ปุ่ม **Run anyway** จะปรากฏ
3. คลิกมัน installer จะรับช่วงต่อจากตรงนั้น

![เมื่อคลิก "More info" ปุ่ม "Run anyway" จะปรากฏข้างๆ "Don't run"](fig-smartscreen-run-anyway.svg)

ทางอ้อมทั้งหมดเพิ่มประมาณ 3 นาที — ส่วนใหญ่เป็นเรื่องจิตใจ ไม่ใช่จำนวนคลิกจริง จากตรงนี้ไป เส้นทางนี้กับเส้นทาง 1 มาบรรจบกัน

## การติดตั้ง macOS — ขั้นตอนคลิกขวาที่ข้ามไม่ได้ {#path-macos}

ไม่มีหน้าจอน้ำเงินบน Mac แต่คุณดับเบิลคลิกตอนเปิดครั้งแรกไม่ได้ — [macOS Gatekeeper](https://support.apple.com/en-us/102445) จะบล็อก

ขั้นตอนที่ถูกต้อง:

1. ดาวน์โหลด `.dmg` ลาก Keeply เข้าโฟลเดอร์ Applications
2. เปิด Applications หา Keeply
3. **คลิกขวา → Open** (ไม่ใช่ดับเบิลคลิก)

   ![เมนูคลิกขวาของ macOS Finder โดย "Open" ถูก highlight อยู่ด้านบน](fig-macos-rightclick.svg)

4. dialog ปรากฏ — คลิก "Open"

   ![dialog ยืนยันของ macOS โดยปุ่ม "Open" ถูก highlight](fig-gatekeeper-dialog.svg)

แค่นั้น **เฉพาะการเปิดครั้งแรกที่ต้องทำแบบนี้** — ดับเบิลคลิกใช้ได้ปกติหลังจากนั้น

ทำไมต้องอ้อมในครั้งแรก? Gatekeeper บล็อกการเปิดด้วยดับเบิลคลิกสำหรับ app ใดก็ตามที่ยังไม่เห็นว่า notarized คลิกขวา → Open คือวิธีของ Apple ที่จะบอกว่า "ฉันรู้ว่าฉันกำลังติดตั้งอะไรอยู่ ปล่อยฉันผ่าน"

นี่ไม่ใช่ลักษณะเฉพาะของ Keeply Mac app ใหม่ทุกตัวที่ไม่เคยอยู่ในเครื่องของคุณก็ทำตัวแบบเดียวกันตอนเปิดครั้งแรก

## หลังติดตั้ง — ลากโปรเจกต์แรกเข้าไป {#first-project}

ติดตั้งแล้วยังไม่จบ โปรเจกต์แรกของคุณถูกปกป้องในวันเดียวกัน — นั่นถึงจะจบ

เปิด Keeply กด **New project** เลือกโฟลเดอร์ที่คุณกำลังทำงานอยู่จริงๆ

**ลากอะไรเข้าก่อน**: อะไรก็ตามที่คุณถืออยู่ตอนนี้ที่เสียไปไม่ได้ และคุณยังแก้อยู่เรื่อยๆ ข้อเสนอ สัญญา ไฟล์ออกแบบ deck — อันไหนก็ใช้ได้ อย่าเลือกโฟลเดอร์ที่คุณไม่ได้แตะมาหกเดือน คุณค่าของโฟลเดอร์นั้นคือการ archive ไม่ใช่การปกป้อง คนละเรื่อง

scan แรกใช้เวลา 1 ถึง 2 นาที หลังจากนั้น Keeply เฝ้าดูโฟลเดอร์เบื้องหลัง และ **บันทึกเวอร์ชันอัตโนมัติเมื่อคุณบันทึก** ไม่มีปุ่ม "checkpoint" ให้คุณกดเอง

ตัวอย่างสมมติแต่เป็นปกติ: นักออกแบบลากโฟลเดอร์ pitch ไตรมาส 2 เข้าไปทันทีหลังติดตั้ง scan แรก 2 นาที สามวันต่อมาเขาตระหนักว่าเปลี่ยนสีโลโก้ผิดตอนวันเสาร์ที่แล้ว — ดึงเวอร์ชันก่อนหน้าจากประวัติ ใช้ 20 วินาที

คนที่ใช้โปรเจกต์แรกในวันที่ติดตั้งอยู่ต่อนานกว่าคนที่รอหนึ่งสัปดาห์อย่างมาก

## ติดอยู่? 5 ปัญหาพบบ่อย {#troubleshoot}

| อาการ | วิธีแก้ |
| --- | --- |
| ไม่พบคำสั่ง `winget` | แปลว่า Windows ของคุณยังไม่มี App Installer ใช้เส้นทาง 2 (ดาวน์โหลด .exe) แทน — อย่าฝืน |
| Win 11 บอกว่า "needs administrator" | เปิด PowerShell ใหม่โดย **Run as administrator** |
| Mac บอก "cannot be opened because it is from an unidentified developer" | คลิกขวา → Open (ไม่ใช่ดับเบิลคลิก) ดูหัวข้อ macOS ด้านบน |
| network บริษัทบล็อกการดาวน์โหลด | ใช้คำสั่ง winget แทน — มันผ่าน CDN ของ Microsoft และมักผ่านได้ |
| ติดตั้งแล้วเปิดไม่ได้ | restart หนึ่งครั้ง ยังไม่ได้? ส่งอีเมลที่ [support@keeply.work](mailto:support@keeply.work) |

## สิ่งเดียวที่ต้องจำ

หนึ่งสิ่ง:

**หน้าจอน้ำเงินไม่ใช่คำตัดสิน — มันคือชื่อเสียงที่ยังถูกสร้างอยู่**

คุณไม่ต้องข้ามคำเตือน คุณแค่เลือกเส้นทาง winget ที่คำเตือนไม่เคยโผล่
